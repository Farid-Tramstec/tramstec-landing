How memory works

Before vs. after TramsTec.

The difference is not marginal. Without memory, your agent starts from zero every message. With TramsTec, it remembers and reasons across the entire history.

Without memory

  • "Hi, how can I help?" to returning customers
  • Orders buried in chat, never extracted
  • Zero context from past quotes
  • Impossible to search "what did I offer Pedro?"
  • Agent repeats questions it already asked
  • More tokens (you re-feed context every time)

With TramsTec

  • "Hi María, want to repeat the 2kg tortillas?"
  • Orders auto-extracted and structured
  • Quotes and proposals indexed by intent
  • Semantic vector search in <50ms
  • Real continuity across conversations
  • Your agent only asks what it needs
LATAM workflows

Real use cases, not demos.

Each workflow chains several MCP tools. Your agent orchestrates them. TramsTec handles state.

🛒

Retail order with payment

  1. Customer asks for products via WhatsApp
  2. Agent looks up customer + catalog + inventory
  3. Creates draft order and generates payment link (Stripe / Mercado Pago / SPEI)
  4. PSP webhook confirms payment → agent confirms order and notifies the customer

customer_lookup · product_search · inventory_check · order_create_draft · payment_link_create · order_confirm · whatsapp_send_message

💰

Stage-aware collections

  1. Look up customer + pending payment status
  2. Generate SPEI instructions with a unique reference
  3. Send reminder using a Meta-approved template
  4. If no response, escalate to human with context

customer_lookup · payment_status_check · spei_instructions_generate · whatsapp_send_template · handoff_to_human

🧾

Mexico CFDI invoice

  1. Customer requests invoice in chat
  2. Agent looks up customer + referenced payment status
  3. Generates CFDI 4.0 via Facturapi (org's RFC)
  4. Sends signed PDF over WhatsApp media

customer_lookup · payment_status_check · invoice_create · whatsapp_send_media

🛡

Auto opt-out (LFPDPPP)

  1. Inbound message detects "BAJA" / "STOP" / "UNSUBSCRIBE"
  2. Agent records a consent event with auditable evidence
  3. Any future send to this contact is runtime-blocked
  4. Audit log preserves the evidence indefinitely

consent_update
