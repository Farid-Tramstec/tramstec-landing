Cómo funciona la memoria

Antes vs. después de TramsTec.

La diferencia no es marginal. Sin memoria, tu agente arranca de cero en cada mensaje. Con TramsTec, recuerda y razona sobre todo el historial.

Sin memoria

  • "Hola, ¿en qué puedo ayudarte?" a clientes recurrentes
  • Pedidos enterrados en el chat, sin extraer
  • Cero context de cotizaciones pasadas
  • Imposible buscar "lo que le ofrecí a Pedro"
  • Agente repite preguntas que ya respondió
  • Más tokens (le pasas todo el contexto cada vez)

Con TramsTec

  • "Hola María, ¿quieres repetir las 2kg de tortillas?"
  • Pedidos extraídos y estructurados automáticamente
  • Cotizaciones y propuestas indexadas por intent
  • Vector search semántico en <50ms
  • Continuidad real entre conversaciones
  • Tu agente solo pide lo que necesita
Workflows LATAM

Casos reales, no demos.

Cada workflow combina varias tools MCP. Tu agente las orquesta. TramsTec se encarga del estado.

🛒

Pedido retail con pago

  1. Cliente pide productos por WhatsApp
  2. Agente busca cliente + catálogo + inventario
  3. Crea orden draft y genera link de pago (Stripe / Mercado Pago / SPEI)
  4. Webhook PSP confirma pago → agente confirma orden y notifica al cliente

customer_lookup · product_search · inventory_check · order_create_draft · payment_link_create · order_confirm · whatsapp_send_message

💰

Cobranza con etapas

  1. Busca cliente + estado de pagos pendientes
  2. Genera instrucciones SPEI con referencia única
  3. Manda recordatorio con template Meta-approved
  4. Si no responde, escala a humano con contexto

customer_lookup · payment_status_check · spei_instructions_generate · whatsapp_send_template · handoff_to_human

🧾

Factura CFDI México

  1. Cliente solicita factura en chat
  2. Busca cliente + status del pago referenciado
  3. Genera CFDI 4.0 vía Facturapi (RFC de la org)
  4. Envía PDF firmado por WhatsApp

customer_lookup · payment_status_check · invoice_create · whatsapp_send_media

🛡

Opt-out automático (LFPDPPP)

  1. Inbound message detecta "BAJA" / "STOP" / "DESUSCRIBIR"
  2. Agente registra consent event con evidencia auditada
  3. Cualquier send futuro a ese contacto queda bloqueado en runtime
  4. Audit log preserva la evidencia indefinidamente

consent_update
